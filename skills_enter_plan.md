# Внедрение архитектурного фундамента (React / Next.js)

Проектирование "движка" для CompetitorIQ с использованием экспертных скиллов (`senior-frontend`, `react-nextjs-development`, `react-patterns`, `react-ui-patterns`, `react-state-management`, `react-best-practices`). Данный этап предваряет активную работу с UI-компонентами и гарантирует производительность и масштабируемость портала.

## User Review Required

> [!IMPORTANT]
> **Выбор State Manager'а:** Предлагаю использовать **Zustand** вместо Redux Toolkit.
> **Причина:** Zustand идеально подходит для дашбордов. Он не требует сложного бойлерплейта, работает молниеносно, легко интегрируется с React хуками и не вызывает лишних ререндеров, что критично для D3/Three.js визуализаций.

## Proposed Changes

Внедрение будет проходить по очереди, в 4 последовательных шага.

### Шаг 1: Контракты и Типизация (`senior-frontend`) [✅ ВЫПОЛНЕНО]

Закладываем фундамент из TypeScript-интерфейсов, чтобы исключить `any` и магические строки.
- Создадим `types/competitor.ts`: Описание профиля конкурента (ID, имя, метрики по 5 видам анализа). [✅]
- Создадим `types/analysis.ts`: Enum'ы и типы для видов анализа (Финансовый, Маркетинговый, Активность, Динамика, Ассортиментный). [✅]

### Шаг 2: Глобальное состояние (`react-state-management`) [✅ ВЫПОЛНЕНО]

Проектируем центральную нервную систему дашборда на базе Zustand.
- Создадим `store/useCompetitorStore.ts`: [✅]
  - `activeAnalysisMode`: Текущий выбранный режим анализа. [✅]
  - `selectedCompetitorId`: Конкурент, выбранный на 3D-графе для детального обзора. [✅]
  - `hiddenCompetitors`: Массив отключенных конкурентов (через тумблеры). [✅]
  - `timePeriod`: Выбранный период для метрик. [✅]

### Шаг 3: Архитектура Server / Client (`react-nextjs-development`) [✅ ВЫПОЛНЕНО]

Разделяем ответственность между сервером и клиентом (App Router Next.js 15).
- Сервер: `app/layout.tsx` и `app/page.tsx` будут отвечать за начальную загрузку структуры. [✅]
- Клиент: Создадим "Client-wrapper" компоненты (например, `components/dashboard/DashboardClient.tsx`/`OverviewClient.tsx`), внутри которых будут крутиться стейт и D3.js графики, с директивой `"use client"`. [✅]

### Шаг 4: Отказоустойчивость (`react-patterns`, `react-ui-patterns`) [✅ ВЫПОЛНЕНО]

Защищаем UI от падений при рендеринге тяжелых данных.
- Создадим `components/ui/ErrorBoundary.tsx`: Кастомный класс для перехвата JS-ошибок в графиках (чтобы упал только один график, а не весь экран). [✅]
- Настроим `Suspense` и скелетоны загрузки для асинхронных компонентов (`components/ui/LoadingSkeleton.tsx`). [✅]

### Шаг 5: Обновление Project_skills [✅ ВЫПОЛНЕНО]
- Интегрируем новые скиллы в матрицу `Project_skills` (добавление в раздел "Архитектура и логика" и маппинг на Фазу 1). [✅]
