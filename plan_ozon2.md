### 🎯 Ценность для портала

Эта выгрузка — **золотая жила** для конкурентного анализа. В одном файле собраны ВСЕ данные, необходимые для новой матрицы продаж:

1. **Коммерческие метрики** — Выручка (K), Штуки (M), ASP (N), Buyout (P)
2. **Воронка Ozon** — Показы (X), Просмотры карточки (Z), CTR (AA=Z/X), В корзину (AC), ДРР (AI)
3. **Группировка** — Продавец/Конкурент (D) и Бренд (E) — две оси матрицы
4. **Размерная сетка** — извлекается из названия (A) → Размер (F) → Футы (G)

> [!IMPORTANT]
> Данные идеально ложатся на новую архитектуру из задания (единая интерактивная матрица с drill-down). Это **реальный источник данных** для замены текущих mock-данных в портале.

### 🚫 Фильтрация «неподходящих товаров»

Менеджер просит убрать:

| Что убрать | Логика фильтрации | Найдено в файле |
|-----------|-------------------|----------------|
| **Настольный футбол** | `название CONTAINS 'футбол'` | ~8 товаров (строки 53, 56, 71, 97, 114, 137, 166, 169) |
| **Трансформер** | `название CONTAINS 'трансформер'` | ~5 товаров (строки 51, 58, 59, 69, 108) |
| **Хоккей (без аэро-)** | `название CONTAINS 'хоккей' AND NOT CONTAINS 'аэро'` | ~3 товара (строки 49, 52, 193 — «Хоккей турнирный» без слова «аэро» в названии) |

> [!WARNING]
> Фильтрация по категории 3 уровня ненадёжна. Например, «Хоккей турнирный» (I) — это категория Ozon, а аэрохоккей тоже попадает в неё. Нужна фильтрация **по названию товара (A)**, а не по категории.

**Рекомендуемый алгоритм фильтрации:**
```javascript
const isExcluded = (name) => {
  const lower = name.toLowerCase();
  // Убираем настольный футбол
  if (lower.includes('футбол')) return true;
  // Убираем трансформеры
  if (lower.includes('трансформер')) return true;
  // Убираем хоккей без "аэро" 
  if (lower.includes('хоккей') && !lower.includes('аэро')) return true;
  return false;
};
```

**После фильтрации останется ~178 релевантных товаров** (аэрохоккей — основная ниша).
